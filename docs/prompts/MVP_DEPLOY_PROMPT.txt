A1. Yeni dosya olustur: frontend/src/config/features.ts

Icerik:
```typescript
/**
 * MVP Feature Flags
 * Set MVP_MODE = false to enable all features
 */
export const MVP_MODE = true

export const FEATURES = {
  mint: true,            // Always on
  race: true,            // Always on
  leaderboard: true,     // Always on
  treehouse: true,       // Always on
  guide: true,           // Always on

  shop: !MVP_MODE,       // Hide in MVP
  training: !MVP_MODE,   // Hide in MVP
  evolution: !MVP_MODE,  // Hide in MVP
  miniGames: !MVP_MODE,  // Hide in MVP
  quests: !MVP_MODE,     // Hide in MVP
  grandPrix: !MVP_MODE,  // Hide in MVP
  tacticRace: !MVP_MODE, // Hide in MVP
  spectate: !MVP_MODE,   // Hide in MVP
  replay: !MVP_MODE,     // Hide in MVP
  profile: !MVP_MODE,    // Hide in MVP
  accessories: !MVP_MODE,// Hide in MVP
  cosmetics: !MVP_MODE,  // Hide in MVP
} as const
```

A2. Layout.tsx navigasyonu MVP'ye gore duzelt
Dosya: frontend/src/components/Layout.tsx

NAV_ITEMS dizisinde Shop linkini kosullu yap.
Ust kisma import ekle:
  import { FEATURES } from '../config/features'

NAV_ITEMS dizisini bul ve degistir. Mevcut dizi:
  - Home, Treehouse, Race, Shop, Leaderboard seklinde

Shop'u filtrele. NAV_ITEMS tanimini bul ve sonrasina filter ekle:
  .filter(item => {
    if (item.path === '/shop' && !FEATURES.shop) return false
    return true
  })

Mobile drawer'da da Profile ve Mint linklerini kontrol et:
  - Profile linkini FEATURES.profile ile kosullu yap
  - Mint her zaman gorunsun

A3. App.tsx route'larini MVP'ye gore duzelt
Dosya: frontend/src/App.tsx

Ust kisma import ekle:
  import { FEATURES } from './config/features'

Asagidaki route'lari kosullu yap veya redirect'e cevir:
  - /shop → FEATURES.shop ? <Shop /> : <Navigate to="/" />
  - /spectate → FEATURES.spectate ? <Spectate /> : <Navigate to="/" />
  - /replay/:id → FEATURES.replay ? <RaceReplay /> : <Navigate to="/" />
  - /profile → FEATURES.profile ? <Profile /> : <Navigate to="/" />

/mint, /treehouse, /race, /race/:id, /leaderboard, /guide → her zaman aktif kalsin.
Navigate import'unu react-router-dom'dan ekle (zaten olabilir).

Proje kokunde (~/slug-rush/) README.md olustur. Icerik:

```markdown
# Sloth Rush

> Wake up. Race hard. Nap later.

Free-to-play sloth racing game built natively on **Base L2**. Mint a Free Sloth, race in deterministic simulations, earn ZZZ Coins, and upgrade your racer.

## Features

- **Free Sloth Minting** — Gasless, one per wallet
- **Deterministic Racing** — Seeded PRNG engine, provably fair results recorded on-chain
- **4 Race Formats** — Exhibition (free), Standard, Tactic, Grand Prix
- **Training & Evolution** — 7 tiers, 3 evolution paths (Caffeine, Hibernate, Dreamwalk)
- **Economy** — ZZZ Coins, daily bonuses, quests, seasonal rewards
- **Leaderboards** — Career stats, ranked leagues, Hall of Fame

## Tech Stack

| Layer | Technology |
|-------|-----------|
| Frontend | React 19 + TypeScript + Vite + Tailwind CSS 4 |
| Backend | Express + PostgreSQL |
| Blockchain | Base L2 (Sepolia), Solidity, Hardhat |
| Wallet | RainbowKit + Wagmi + Viem |
| Race Engine | Deterministic simulation (mulberry32 PRNG) |

## Architecture

```
sloth-rush/
├── frontend/        # React SPA (Vercel)
├── backend/         # Express API + PostgreSQL (Railway)
├── contracts/       # Solidity smart contracts (Base L2)
└── docs/            # Light paper, prompts
```

**Hybrid Model:** High-integrity operations (minting, upgrades, race hashes) live on-chain. High-frequency gameplay (racing, training, economy) runs off-chain for speed and zero gas costs.

## Smart Contracts

- **FreeSloth.sol** — ERC-721 NFT, one per wallet, burnable
- **Sloth.sol** — ERC-721 with on-chain stats (SPD, ACC, STA, AGI, REF, LCK)
- **SlothRush.sol** — Upgrade (burn FreeSloth + mint Sloth), race result recording

## Local Development

### Prerequisites
- Node.js 18+
- PostgreSQL
- npm

### Setup

```bash
# Install dependencies
npm install

# Start backend (terminal 1)
cd backend && npm run dev

# Start frontend (terminal 2)
cd frontend && npm run dev
```

Frontend runs on http://localhost:5173, backend on http://localhost:3001.

### Environment Variables

**Backend (.env):**
```
DATABASE_URL=postgresql://localhost:5432/sloth_rush
NODE_ENV=development
PORT=3001
```
