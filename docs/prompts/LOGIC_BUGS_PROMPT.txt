### A5. GP Tier Gate Ekle (HATA 12)
Dosya: backend/src/routes/race.ts — /join endpoint'i

GP yarislarina katilirken tier kontrolu ekle:

GP join sirasinda:
- Silver GP (format === 'gp' ve race.entry_fee <= 150): Minimum snail (tier >= 1) gerekli
- Gold GP (format === 'gp' ve race.entry_fee > 150): Minimum elite (tier >= 2) gerekli

const slug = await getOne("SELECT tier, type FROM slugs WHERE id = $1", [snailId]);
if (race.format === 'gp') {
  if (race.entry_fee > 150 && (slug.tier || 0) < 2) {
    res.status(400).json({ error: "Gold GP requires Elite snail (Tier 2+)" }); return;
  }
  if (slug.type === 'free_slug') {
    res.status(400).json({ error: "GP requires at least a Snail" }); return;
  }
}

Frontend'de: RaceLobby.tsx'de GP format secildiginde, free slug'lari snail listesinden filtrele. Gold GP secildiginde tier < 2 snail'leri filtrele.

## BOLUM B: STAT & TRAINING DUZELTMELERI

### B1. Training Limitini Haftalik Yap (HATA 4 — KRITIK)
Dosya: backend/src/routes/slug.ts — /train endpoint'i

Mevcut: Gunluk 2 training limiti (per day).
Yeni: Haftalik 2 training limiti (per week per snail).

Training limit sorgusunu degistir:
Eski: WHERE snail_id = $1 AND DATE(started_at) = CURRENT_DATE
Yeni: WHERE snail_id = $1 AND started_at >= date_trunc('week', CURRENT_TIMESTAMP)

Limit sayisi ayni kalsin:
- Free slug: haftada 1
- Snail: haftada 2

Frontend'de de (Stable.tsx) training limit mesajini guncelle:
Eski: "(2 per day)" veya benzer
Yeni: "X/2 this week" seklinde goster

### B2. Mini-Game Limitini Wallet Bazli Yap (HATA 5 — KRITIK)
Dosya: backend/src/routes/slug.ts — /mini-game endpoint'i

Mevcut: Snail bazli gunluk limit (her snail icin ayri 5 hak).
Yeni: Wallet bazli gunluk limit (toplam 5 hak).

Limit sorgusunu degistir:
Eski: SELECT count FROM daily_minigame_plays WHERE snail_id = $1 AND play_date = $2
Yeni:
SELECT COALESCE(SUM(dmp.count), 0) as total_count
FROM daily_minigame_plays dmp
JOIN slugs s ON dmp.snail_id = s.id
WHERE s.wallet = $1 AND dmp.play_date = $2

dailyLimit = 5 (herkes icin ayni, free slug farki kaldiriliyor)

### B3. Stat Cap Guvenlik Kontrolu (HATA 7)
Dosya: backend/src/routes/slug.ts

Training claim ve mini-game stat gain hesaplarinda:
Mevcut: Math.min(gain, cap - current)
Yeni: Math.min(gain, Math.max(0, cap - (current || 0)))

Bu degisikligi su yerlere uygula:
1. /claim-training endpoint'inde stat gain hesabi
2. /mini-game endpoint'inde stat gain hesabi

Bu Math.max(0, ...) eklentisi current'in null/undefined olmasi durumunda negatif deger olusmasini onler.
